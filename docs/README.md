## üìù Contributing to Documentation

Documentation contributions are welcome! See [Contributing Guide](../CONTRIBUTING.md) for details.

### Documentation Standards

- **Markdown**: Use GitHub Flavored Markdown
- **Structure**: Include introduction, sections, examples, and related links
- **Length**: Aim for comprehensive coverage (400+ lines for format guides)
- **Examples**: Provide practical, working examples
- **Cross-references**: Link to related documents
- **Updates**: Keep "Last Updated" dates current

### Adding New Documentation

1. Create document in appropriate directory
2. Follow existing format and style
3. Add entry to this index
4. Cross-reference from related documents
5. Update README.md in project root if applicable
6. Submit pull request

### Building Documentation

```bash
# Generate API documentation (KDoc)
./gradlew dokkaHtml

# View in browser
open shared/build/dokka/html/index.html
```

### Finding Documentation

```bash
# Search all documentation
grep -r "search term" docs/

# List all format guides
ls docs/user-guide/formats/*.md

# Count documentation lines
find docs/ -name "*.md" -exec wc -l {} + | tail -1
```
